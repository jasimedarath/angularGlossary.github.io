declarations

  • This array is used to declare components, directives, and pipes that belong to the module.
  • Angular needs to know about these declarations so it can recognize and use them in templates.

imports

  • This array is used to import other modules that export components, directives, and pipes needed by the components in this module.
  • By importing a module, all the declared components, directives, and pipes of the imported module become available to this module.

providers

  • This array is used to define the services that the injector will provide.
  • These services can be injected into components, directives, and other services within the module.

bootstrap

  • This array is used to specify the root component that Angular should bootstrap when the application is started.
  • Typically, this is only used in the root module (e.g., AppModule).

exports

  • Specifies a set of components, directives, and pipes that can be used in the templates of other modules.

entryComponents

  • Specifies a set of components that are compiled when the module is defined, rather than when they are referenced in a template.
  • Commonly used for dynamically loading components.

schemas

  • Specifies a set of schemas that declare elements to be recognized by the Angular compiler.

id

  • Provides a unique identifier for the module, useful for debugging or analytics.
