CanActivate

  • Purpose: Determines if a route can be activated.
  • Usage: Often used for authentication checks.

CanActivateChild

  • Purpose: Determines if the children of a route can be activated.
  • Usage: Often used to apply guard logic to child routes.
{{ activateChild }}

CanDeactivate

  • Purpose: Determines if a route can be deactivated.
  • Usage: Used to prevent navigation away from the current route, such as when there are unsaved changes.
{{ deactivate }}

Resolve

  • Purpose: Pre-fetches data before activating a route.
  • Usage: Used to ensure that necessary data is loaded before the route is activated.
{{ resolve }}

CanLoad

  • Purpose: Determines if a module can be loaded asynchronously.
  • Usage: Used to control access to lazy-loaded modules.
{{ load }}
