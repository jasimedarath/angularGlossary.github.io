Next.js Rendering Strategies

Next.js provides multiple rendering strategies to optimize performance and user experience: SSR, SSG, ISR, and CSR.

Server-Side Rendering (SSR)

SSR renders pages on the server for each request, ensuring fresh data.

  • Pages rendered on every request
  • Always fresh, up-to-date data
  • Use cache: 'no-store' to opt into SSR
  • Good for personalized content
  • SEO-friendly with fresh content
  • Slower initial load compared to static
{{ssrBasics}}

Static Site Generation (SSG)

SSG generates pages at build time for maximum performance.

  • Pages generated during build
  • Served as static HTML
  • Ultra-fast loading times
  • Perfect for blogs and documentation
  • Use generateStaticParams for dynamic routes
  • Data is stale until next build
{{ssgBasics}}

Incremental Static Regeneration (ISR)

ISR combines static generation with periodic updates.

  • Static pages with periodic revalidation
  • Best of both SSG and SSR
  • Set revalidation period with revalidate
  • On-demand revalidation with revalidatePath
  • Perfect for e-commerce and content sites
  • Scales to millions of pages
{{isrBasics}}

Client-Side Rendering (CSR)

CSR renders content in the browser using JavaScript.

  • Rendering happens in the browser
  • Use 'use client' directive
  • Good for interactive, user-specific UI
  • Access to browser APIs
  • Can use React hooks
  • Slower initial page load
{{csrBasics}}

Rendering Methods Comparison

Choose the right rendering strategy based on your content and requirements.

  • SSG: Best performance, stale data
  • ISR: Great performance, periodic freshness
  • SSR: Always fresh, slower load
  • CSR: Interactive, browser-only

Generating Static Params

generateStaticParams defines which dynamic routes to pre-render at build time.

  • Replaces getStaticPaths from Pages Router
  • Generates static pages for dynamic routes
  • Can be nested with parent params
  • Runs at build time
  • Returns array of param objects
  • Enables SSG for dynamic routes
{{generateStaticParams}}

Revalidation Strategies

Control when and how your static content is regenerated.

  • Time-based: Set revalidate interval
  • On-demand: Trigger with revalidatePath/revalidateTag
  • Tags: Group related content for bulk revalidation
  • Works with ISR
  • Efficient cache invalidation
  • No-store for SSR
{{revalidation}}

Streaming with Suspense

Streaming allows you to progressively render UI as data becomes available.

  • Fast Time to First Byte (TTFB)
  • Progressive rendering with Suspense
  • Instant loading states
  • Better perceived performance
  • SEO-friendly HTML streaming
  • Works with Server Components
{{streaming}}
