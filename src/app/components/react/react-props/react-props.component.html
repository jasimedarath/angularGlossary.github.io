React Props

Props (short for properties) are how you pass data from parent components to child components in React. They are read-only and help make components reusable and configurable.

Understanding Props

Props are arguments passed to React components, similar to function parameters.

  • Props are passed from parent to child components
  • Props are read-only (immutable)
  • Props can be any JavaScript value: strings, numbers, objects, arrays, functions
  • Use curly braces for non-string values
  • Props enable component reusability

Props Destructuring

Destructuring makes it easier to access props by extracting them directly in the function parameters.

  • Extract props in function parameters
  • Cleaner and more readable code
  • Can use nested destructuring for nested objects
  • Use rest operator (...) to collect remaining props
  • Combine with default values
{{propsDestructuring}}

Default Props

Default props provide fallback values when props are not provided by the parent.

  • Use ES6 default parameters in function signatures
  • Provides fallback values for undefined props
  • Makes components more robust
  • Can use complex default values (objects, arrays)
  • Combined with destructuring for clean syntax
{{defaultProps}}

Props Validation with PropTypes

PropTypes provide runtime type checking for React props (use TypeScript for better type safety).

  • Runtime validation of prop types
  • Helps catch bugs during development
  • Documents component API
  • Use .isRequired for mandatory props
  • Supports complex types (arrays, objects, shapes)
  • Consider TypeScript for compile-time type checking
{{propTypes}}

The Children Prop

Children is a special prop that contains the content between opening and closing component tags.

  • Special prop automatically passed by React
  • Contains nested content/components
  • Can be any valid JSX: text, elements, components
  • Use React.Children API to manipulate children
  • Enables composition patterns
{{childrenProp}}

Spreading Props

Spread operator allows passing all props or selected props to child components efficiently.

  • Pass all props at once with {...props}
  • Useful for wrapper components
  • Combine with destructuring for selective passing
  • Can override spread props by placing them after
  • Be careful not to pass unnecessary props
{{propsSpread}}

Passing Functions as Props

Function props enable child components to communicate with parent components (callbacks).

  • Pass functions to handle events in child components
  • Enable child-to-parent communication
  • Common pattern for form inputs and buttons
  • Use arrow functions to pass arguments
  • Name function props with "on" prefix (onClick, onSubmit)
