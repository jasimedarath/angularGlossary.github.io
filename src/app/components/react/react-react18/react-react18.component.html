React 18 Features

Explore the latest features introduced in React 18, including concurrent rendering, transitions, and server components.

Concurrent Rendering

Concurrent Rendering allows React to work on multiple tasks simultaneously and prioritize urgent updates.

  • Interruptible rendering process
  • Prioritize user interactions
  • Better perceived performance
  • New root API with createRoot
{{concurrentRendering}}

Transitions & Deferred Values

useTransition and useDeferredValue help distinguish urgent from non-urgent updates.

  • useTransition marks updates as transitions
  • useDeferredValue defers value updates
  • Keep UI responsive during heavy updates
  • Show pending states
{{transitions}}

Suspense for Data Fetching

Suspense allows components to "wait" for something before rendering.

  • Code splitting with React.lazy
  • Data fetching (experimental)
  • Nested loading states
  • Better loading UX
{{suspense}}

Automatic Batching

React 18 automatically batches all state updates, even in promises, timeouts, and event handlers.

  • Fewer renders, better performance
  • Works in all contexts
  • flushSync to opt-out
  • Backwards compatible
{{automaticBatching}}

React Server Components

Server Components run only on the server, reducing bundle size and enabling direct backend access.

  • Zero bundle size for server components
  • Direct database/file system access
  • Automatic code splitting
  • Better performance
{{serverComponents}}

New Hooks in React 18

React 18 introduces new hooks for specific use cases.

  • useId: Generate unique IDs for accessibility
  • useSyncExternalStore: Subscribe to external stores
  • useInsertionEffect: CSS-in-JS libraries
{{newHooks}}
