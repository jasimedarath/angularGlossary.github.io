React Testing

Learn how to test React applications using Jest and React Testing Library for reliable, maintainable tests.

Jest Testing Framework

Jest is a JavaScript testing framework with built-in assertions, mocking, and code coverage.

  • Zero configuration for most projects
  • Snapshot testing
  • Mock functions and modules
  • Code coverage reports
{{jestBasics}}

React Testing Library

React Testing Library helps test React components the way users interact with them.

  • Focus on behavior, not implementation
  • Query elements like users do
  • Encourage accessible markup
  • Works with DOM Testing Library
{{reactTestingLibrary}}

Query Methods

Different query methods for finding elements in your tests.

  • getBy: Throws error if not found
  • queryBy: Returns null if not found
  • findBy: Async, waits for element
  • getAllBy/queryAllBy/findAllBy: Multiple elements
{{queryMethods}}

Testing Async Code

Test asynchronous operations like API calls, loading states, and timers.

  • findBy queries for async elements
  • waitFor for complex conditions
  • Mock fetch/API calls
  • Test loading and error states
{{asyncTesting}}

Testing Custom Hooks

Use renderHook to test custom hooks in isolation.

  • Test hook logic independently
  • Test with different props
  • Test hook state updates
  • Use act() for state changes
{{hooksTest}}

Mocking and Context

Mock modules, provide context, and use fake timers in tests.

  • Mock API modules
  • Provide context in tests
  • Mock timers for debounce/throttle
  • Custom render with providers
{{mockingContext}}

Testing Best Practices

✓ Do's

  • Test user behavior, not implementation
  • Use accessible queries (getByRole)
  • Test error states and edge cases
  • Keep tests isolated and independent
  • Use meaningful test descriptions

✗ Don'ts

  • Don't test implementation details
  • Don't access component state directly
  • Don't rely on internal structure
  • Avoid testing third-party libraries
