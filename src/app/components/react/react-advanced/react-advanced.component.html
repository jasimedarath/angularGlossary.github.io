Advanced React Patterns

Learn advanced React patterns and techniques for building scalable, maintainable applications.

Higher-Order Components (HOC)

HOCs are functions that take a component and return a new enhanced component.

  • Reuse component logic
  • Add props or behavior
  • Abstract common patterns
  • Legacy pattern (hooks are preferred now)
{{higherOrderComponents}}

Render Props Pattern

Render Props use a function prop to share code between components.

  • Component receives function as prop
  • Function returns React elements
  • Share stateful logic
  • Custom hooks are modern alternative
{{renderProps}}

React Portals

Portals render children into a DOM node outside the parent component hierarchy.

  • Render modals, tooltips, dropdowns
  • Break out of parent container
  • Maintain React tree behavior
  • Events bubble up React tree
{{portals}}

Error Boundaries

Error Boundaries catch JavaScript errors in component tree and display fallback UI.

  • Only available as class components
  • Catch errors during render
  • Display fallback UI
  • Log errors for debugging
{{errorBoundaries}}

Refs & Forward Refs

Refs provide access to DOM elements and store mutable values without causing re-renders.

  • Access DOM elements directly
  • Store mutable values
  • Forward refs to child components
  • Doesn't trigger re-renders
{{refs}}

Compound Components Pattern

Compound Components are components that work together to form a complete UI.

  • Share implicit state
  • Flexible API
  • Encapsulate complex behavior
  • Used in UI libraries (Tabs, Select, Accordion)
{{compoundComponents}}
