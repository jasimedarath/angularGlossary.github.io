JSX - JavaScript XML

JSX is a syntax extension for JavaScript that looks similar to HTML. It allows you to write UI components in a more intuitive way by mixing HTML-like syntax with JavaScript logic.

What is JSX?

JSX stands for JavaScript XML. It's a syntax extension that allows you to write HTML-like code in JavaScript files. JSX gets compiled to regular JavaScript function calls.

  • JSX produces React "elements"
  • It's not required but makes code more readable
  • You can embed JavaScript expressions using curly braces
  • JSX is closer to JavaScript than HTML

React Fragments

Fragments let you group multiple children without adding extra nodes to the DOM.

  • Use <Fragment> or short syntax <>
  • Helps maintain clean DOM structure
  • Can have keys when rendering lists
  • No extra wrapper div in the DOM
{{jsxFragments}}

JavaScript Expressions in JSX

You can embed any JavaScript expression in JSX by wrapping it in curly braces.

  • Arithmetic operations
  • Function calls
  • Ternary operators for conditional rendering
  • Logical && for conditional display
  • Array methods like map, filter
{{jsxExpressions}}

JSX Attributes

JSX uses camelCase for attributes and some have different names than HTML.

  • class → className
  • for → htmlFor
  • Event handlers use camelCase: onClick, onChange
  • Style attribute accepts a JavaScript object
  • Custom data attributes: data-*
{{jsxAttributes}}

JSX Children

JSX elements can contain children - other elements, text, or expressions.

  • Children can be nested JSX elements
  • String literals as children
  • JavaScript expressions as children
  • Arrays of elements (with keys)
  • Children as props for composition
{{jsxChildren}}

JSX Best Practices

✓ Do's

  • Always provide keys when rendering lists
  • Keep JSX readable with proper indentation
  • Extract complex JSX into separate components
  • Use meaningful component and variable names
  • Wrap multi-line JSX in parentheses
  • Use fragments to avoid extra wrapper divs

✗ Don'ts

  • Don't use index as key unless absolutely necessary
  • Don't create components inside render methods
  • Don't forget to close self-closing tags
  • Don't use quotes for JavaScript expressions in attributes
  • Avoid deeply nested JSX (extract to components)
