React Conditional Rendering

Conditional rendering in React allows you to render different components or elements based on certain conditions, similar to how conditions work in JavaScript.

Ternary Operator (? :)

The ternary operator is the most common way to conditionally render elements inline.

  • Syntax: condition ? trueValue : falseValue
  • Inline conditional rendering
  • Can render components or values
  • Keep it simple for readability
{{ternaryOperator}}

Logical AND (&&) Operator

Use && operator when you want to render something only if a condition is true.

  • Renders right side if left side is truthy
  • Returns nothing if condition is false
  • Simpler than ternary for one-sided conditions
  • Beware of falsy values like 0
{{logicalAnd}}

If-Else Statements

Use if-else statements outside JSX for complex conditions.

  • Multiple return statements
  • Guard clauses for early returns
  • Better for complex logic
  • More readable than nested ternaries
{{ifElseStatements}}

Switch Statement

Switch statements handle multiple conditions, but object mapping is often better.

  • Handle multiple distinct values
  • Use object mapping for cleaner code
  • Requires break statements
  • Consider lookup objects instead
{{switchCase}}

Nullish Coalescing (??)

Nullish coalescing (??) provides default values for null/undefined, and optional chaining (?.) safely accesses nested properties.

  • ?? returns right side if left is null/undefined
  • Different from || which checks truthiness
  • ?. optional chaining for safe property access
  • Prevents "cannot read property" errors
{{nullishCoalescing}}

Conditional Classes & Styles

Apply CSS classes and styles conditionally based on component state or props.

  • Template literals for dynamic classes
  • classnames library for complex conditions
  • Inline style objects with conditions
  • Keep styling logic separate when complex
{{conditionalClasses}}

Render Props Pattern

Render props allow passing rendering logic as a function prop.

  • Share logic between components
  • Function receives data, returns JSX
  • More flexible than HOCs
  • Hooks are often better alternative
{{renderProps}}
