Getting Started with Angular

Learn how to set up your development environment and create your first Angular application.

Prerequisites

Before starting with Angular, ensure you have the required tools and knowledge.

  • Node.js 18.13 or newer (LTS recommended)
  • npm or yarn package manager
  • Code editor (VS Code recommended)
  • Basic JavaScript/TypeScript knowledge
{{prerequisites}}

Installing Angular CLI

Angular CLI is the command-line interface tool for Angular development.

  • Install globally using npm
  • Provides ng commands
  • Generates components, services, etc.
  • Manages build and development server
{{installation}}

Creating a New Application

ng new command creates a new Angular workspace and application.

  • Interactive prompts for configuration
  • Choose routing and styling options
  • Generates complete project structure
  • Installs dependencies automatically
{{createApp}}

Project Structure

Understanding the Angular project structure helps you navigate and organize your code.

  • src/ - Source code directory
  • app/ - Application components and logic
  • angular.json - Angular CLI configuration
  • tsconfig.json - TypeScript configuration
{{projectStructure}}

Essential CLI Commands

Angular CLI provides commands for all development tasks.

  • ng serve - Start development server
  • ng generate - Create components/services
  • ng build - Build for production
  • ng test - Run unit tests
{{basicCommands}}

Running Your Application

Development server provides live reload and debugging capabilities.

  • Automatic reload on file changes
  • Source maps for debugging
  • Error messages in browser
  • Multiple serving options
{{runningApp}}

Creating Your First Component

Components are the building blocks of Angular applications.

  • Use ng generate component command
  • Creates TypeScript, HTML, CSS, and test files
  • Standalone components by default (Angular 18+)
  • Use selector to include in templates
{{firstComponent}}

Understanding package.json

package.json defines project dependencies and npm scripts.

  • Dependencies for runtime
  • DevDependencies for development
  • Scripts for common tasks
  • Project metadata
{{packageJson}}

Common Issues and Solutions

Troubleshooting common Angular development issues.

  • Port conflicts
  • Node version issues
  • Module not found errors
  • Build and compilation errors
{{commonIssues}}
