Angular Material

What is Angular Material?

Angular Material is a UI component library that implements Google's Material Design specifications. It provides a set of reusable, well-tested, and accessible UI components for Angular applications.

Key Features

  • Pre-built Components: 40+ production-ready components
  • Material Design: Follows Google's design system
  • Accessibility: WCAG 2.1 compliant components
  • Responsive: Mobile-first, responsive layouts
  • Theming: Powerful theming and customization
  • TypeScript: Full TypeScript support

  • ‚ö° Fast Development: Pre-built components speed up development
  • üé® Consistent Design: Uniform look and feel across app
  • ‚ôø Accessible: Built-in accessibility features
  • üì± Responsive: Works seamlessly on all devices
  • üîß Customizable: Extensive theming capabilities
  • üìö Well Documented: Comprehensive documentation and examples

Component Categories

  • Form Controls: Input, select, checkbox, radio, slider, datepicker
  • Navigation: Toolbar, sidenav, menu, tabs
  • Layout: Card, expansion panel, stepper, grid list
  • Buttons & Indicators: Button, FAB, badge, progress bar, spinner
  • Popups & Modals: Dialog, snackbar, tooltip, bottom sheet
  • Data Tables: Table, paginator, sort

Angular Material Theming

Theming in Angular Material allows you to customize the colors, typography, and density of your application. Material uses Sass to generate themes that can be applied globally or to specific components.

Theme Structure

  • Primary Palette: Main brand color used throughout app
  • Accent Palette: Secondary color for highlights and actions
  • Warn Palette: Color for warnings and errors
  • Typography: Font families, sizes, and weights
  • Density: Component compactness (0 to -5 scale)

Pre-built Themes

  • indigo-pink.css: Indigo primary, pink accent
  • deeppurple-amber.css: Deep purple primary, amber accent
  • pink-bluegrey.css: Pink primary, blue grey accent
  • purple-green.css: Purple primary, green accent

Color Palettes

  • Material provides 19 pre-defined color palettes
  • Each palette has shades from 50 (lightest) to 900 (darkest)
  • Special shades: A100, A200, A400, A700 (accent variations)
  • Each shade has a corresponding contrast color

üí° Theme Application

Use @include mat.all-component-themes($theme) to apply your theme to all Material components. For better performance in production, use @include mat.all-component-colors($theme) if you only need to change colors.

üéØ Theming Best Practices

  • Define your theme in a separate _theme.scss file
  • Use semantic color names (primary, accent, warn)
  • Test your theme with both light and dark modes
  • Ensure sufficient color contrast for accessibility
  • Use Material's color system instead of custom colors

‚ö†Ô∏è Theme Pitfalls

  • Including core styles multiple times (use @include mat.core() once)
  • Overriding Material styles with !important (use theming instead)
  • Forgetting to test dark theme thoroughly
  • Not considering color contrast ratios

Building Responsive Layouts

Angular Material provides powerful tools for creating responsive layouts that adapt to different screen sizes. Use the CDK Layout module to detect breakpoints and adjust your UI accordingly.

Breakpoint Detection

  • BreakpointObserver: Reactive breakpoint detection service
  • Standard Breakpoints: XSmall, Small, Medium, Large, XLarge
  • Device Types: Handset, Tablet, Web
  • Custom Breakpoints: Define your own media queries

Responsive Patterns

  • Adaptive Sidenav: Side mode on desktop, over mode on mobile
  • Responsive Grid: Dynamic column count based on screen size
  • Conditional Rendering: Show/hide components per device
  • Responsive Tables: Compact view on mobile devices
  • Flexible Toolbar: Adapt toolbar content to screen width

Material Breakpoints

  • XSmall: < 600px (phones in portrait)
  • Small: 600px - 959px (phones landscape, tablets portrait)
  • Medium: 960px - 1279px (tablets landscape, small desktops)
  • Large: 1280px - 1919px (desktops)
  • XLarge: ‚â• 1920px (large desktops)

üí° Mobile-First Approach

Design for mobile screens first, then progressively enhance for larger screens. This ensures your app works well on all devices and improves performance on mobile networks.

üéØ Responsive Design Tips

  • Test on real devices, not just browser DevTools
  • Use BreakpointObserver instead of CSS media queries in components
  • Consider touch targets (minimum 48x48px for buttons)
  • Optimize images for different screen sizes
  • Test landscape and portrait orientations

  • Touch targets too small (< 44x44px)
  • Ignoring landscape orientation

Accessibility in Angular Material

Angular Material components are designed to be accessible by default, implementing WCAG 2.1 Level AA standards. However, proper usage and additional configuration are required to ensure full accessibility.

Built-in Accessibility Features

  • ARIA Labels: Semantic labels for screen readers
  • Keyboard Navigation: Full keyboard support for all components
  • Focus Management: Proper focus indicators and trap
  • Color Contrast: WCAG compliant color ratios
  • Screen Reader Support: Descriptive announcements

Accessibility Tools

  • FocusMonitor: Track and manage element focus
  • LiveAnnouncer: Announce dynamic content to screen readers
  • A11yModule: CDK accessibility utilities
  • InteractivityChecker: Detect focusable elements

Testing Accessibility

  • üéπ Keyboard Only: Navigate entire app without mouse
  • üîä Screen Readers: Test with NVDA, JAWS, VoiceOver
  • üîç Automated Tools: axe, Lighthouse accessibility audits
  • üé® Color Contrast: Check all text meets minimum ratios
  • üîé Zoom Testing: Test at 200% zoom level
{{ accessibility }}

‚úÖ Accessibility Checklist

  • Semantic HTML: Use proper heading hierarchy (h1-h6)
  • ARIA Labels: Provide labels for icon-only buttons
  • Keyboard Access: All interactive elements keyboard accessible
  • Focus Indicators: Visible focus states for all elements
  • Color Contrast: Text meets 4.5:1 ratio minimum
  • Error Messages: Clear, descriptive error feedback
  • Form Labels: Every input has associated label

üí° WCAG Standards

Web Content Accessibility Guidelines (WCAG) 2.1 defines three conformance levels: A (minimum), AA (recommended), and AAA (enhanced). Angular Material targets Level AA compliance.

‚ö†Ô∏è Accessibility Pitfalls

  • Missing aria-labels on icon-only buttons
  • Custom components without proper ARIA attributes
  • Insufficient color contrast in custom themes
  • Dialog focus not managed properly
  • Dynamic content not announced to screen readers

üéØ Performance Tips
